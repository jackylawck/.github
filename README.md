# 🏛️ Centralized Governance & Security Policies

Welcome to the centralized governance repository for the AI Governance and Compliance ecosystem maintained by Jacky Law. 

Operating under the principles of **Governance-as-Code** and aligning with international management system standards (ISO/IEC 42001 AIMS & ISO/IEC 27001 ISMS), this repository serves as the **Single Source of Truth (SSOT)** for all community health files, security disclosures, and operational policies across my public GitHub portfolio.

## 📂 Global Policies Implemented

By utilizing GitHub's default community health file mechanism, the policies housed in this repository automatically apply to all public repositories within this account (unless explicitly overridden by a local repository file).

### 1. [Security & Vulnerability Disclosure Policy](SECURITY.md)
*   **Purpose:** Establishes a strict, private reporting channel for any identified security flaws, data leakage risks, or bypasses in deterministic AI guardrails.
*   **Alignment:** Complies with enterprise-grade incident management and vulnerability disclosure workflows.

### 2. [Community Code of Conduct](CODE_OF_CONDUCT.md)
*   **Purpose:** Sets the baseline for professional, inclusive, and respectful interactions within the ecosystem. 
*   **Approach:** Implements a **Mediation-First** grievance mechanism to resolve technical or conceptual disputes, reflecting human-centred governance practices.
